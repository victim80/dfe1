Volatile data is lost when a machine is turned off/powered down. However, during forensic investigation, investigators need to collect this data to construct a timeline analysis of the incident that occurred.

Volatile Information includes:

  • Hostname, date & time, and time zone
  • Uptime
  • Network information
  • Open ports
  • Open files
  • Mounted filesystem information
  • Loaded kernel modules
  • User events
  • Running processes
  • Swap areas and Disk partition information
  • Kernel messages
